# gpui-table-core Architecture

This document describes the internal architecture of the `gpui-table-core` crate.

## Overview

`gpui-table-core` is the foundation crate providing core traits, types, and the filter infrastructure for the gpui-table system. It defines the abstractions that all other crates in the workspace build upon.

## Module Structure

```
src/
├── lib.rs              # Public API and re-exports
├── filter/             # Filter system
│   ├── mod.rs          # Module exports
│   ├── traits.rs       # Matchable<F>, FilterValuesExt, FilterEntitiesExt
│   ├── value.rs        # FilterValue, Filterable traits
│   ├── config.rs       # FilterConfig, FilterType, FacetedFilterOption
│   ├── wrappers.rs     # FacetedValue<T>, RangeValue<T>, TextValue
│   └── convert.rs      # ToDecimal, ToNaiveDate conversion traits
└── registry.rs         # Metadata registry (GpuiTableShape, ColumnVariant, etc.)
```

## Core Traits

### TableCell

Defines how values are rendered in table cells. Implemented for:
- Primitive types (String, integers, floats, bool)
- Optional types (`Option<T>`)
- Date/time types (chrono - when feature enabled)
- Decimal types (rust_decimal - when feature enabled)

```rust
pub trait TableCell {
    fn render_cell(&self, cx: &WindowContext) -> impl IntoElement;
}
```

### TableRowMeta

The core trait defining table structure. Generated by `#[derive(GpuiTable)]`.

```rust
pub trait TableRowMeta: 'static + Clone + Send + Sync {
    const TABLE_ID: &str;
    const TABLE_TITLE: &str;
    
    fn table_columns() -> Vec<TableColumn>;
    fn cell_value(&self, col_index: usize) -> Option<Box<dyn TableCell>>;
    fn table_filters() -> Vec<FilterConfig>;
}
```

### TableRowStyle

Styling hooks for customizing cell and row rendering.

```rust
pub trait TableRowStyle: TableRowMeta {
    fn render_table_cell(&self, col: &TableColumn, cx: &mut WindowContext) -> impl IntoElement;
    fn render_table_row(&self, cols: &[TableColumn], cx: &mut WindowContext) -> impl IntoElement;
}
```

### Data Loading Traits

```rust
pub trait TableDataLoader {
    fn load_data(&self, offset: usize, limit: usize) -> Vec<Self>;
}

pub trait TableLoader {
    const THRESHOLD: usize;
    fn load_more(&self, cx: &mut Context<Self>) -> Task<Result<TableLoadMore<...>>>;
}
```

## Filter System

### Architecture

The filter system is designed around three concerns:

1. **Filter Configuration** (`filter/config.rs`)
   - `FilterConfig` - Describes a filter's metadata (field name, type, label)
   - `FilterType` - Enum of supported filter types (Text, NumberRange, DateRange, Faceted)
   - `FacetedFilterOption` - Options for faceted filters with label/value/icon

2. **Filter Values** (`filter/value.rs`, `filter/wrappers.rs`)
   - `FilterValue` - Trait for types that can be filter values
   - `Filterable` - Trait linking data types to their filter value types
   - Wrapper types: `TextValue`, `RangeValue<T>`, `FacetedValue<T>`

3. **Filter Matching** (`filter/traits.rs`)
   - `Matchable<F>` - Trait for rows that can be filtered by filter values struct `F`
   - `FilterValuesExt` - Extension methods for filter value structs
   - `FilterEntitiesExt` - Extension methods for filter entity structs

### Type Conversions

The `filter/convert.rs` module provides conversion traits for filter comparisons:

- `ToDecimal` - Convert numeric types to `Decimal` for range comparisons
- `ToNaiveDate` - Convert date types to `NaiveDate` for date range comparisons

## Registry System

The registry (`registry.rs`) uses the `inventory` crate to collect metadata at compile time.

### Key Types

```rust
pub struct GpuiTableShape {
    pub struct_name: &'static str,
    pub columns: &'static [ColumnVariant],
    pub filters: &'static [FilterVariant],
    pub source_path: &'static str,
}

pub struct ColumnVariant {
    pub field_name: &'static str,
    pub field_type: &'static str,
    pub title: &'static str,
    pub width: f32,
    pub sortable: bool,
    pub fixed: ColumnFixed,
}

pub struct FilterVariant {
    pub field_name: &'static str,
    pub filter_type: RegistryFilterType,
}
```

### Usage

The registry is populated by the derive macro and can be iterated at runtime:

```rust
for shape in inventory::iter::<GpuiTableShape> {
    println!("Table: {}", shape.struct_name);
}
```

## Feature Flags

| Feature | Description |
|---------|-------------|
| `chrono` (default) | DateTime type support |
| `rust_decimal` | Decimal type support |
| `fluent` | Internationalization via es-fluent |

## Dependencies

- `gpui` / `gpui-component` - UI framework
- `serde` - Serialization
- `strum` - Enum utilities
- `inventory` - Compile-time registry
- `chrono` (optional) - Date/time types
- `rust_decimal` (optional) - Decimal type
- `es-fluent` (optional) - i18n support
