# gpui-table-component Architecture

This document describes the internal architecture of the `gpui-table-component` crate.

## Overview

`gpui-table-component` provides UI components for table filtering. Each filter type is implemented as a GPUI entity with a consistent interface for integration with the generated filter infrastructure.

## Module Structure

```
src/
├── lib.rs                    # Core trait and re-exports
├── text_filter.rs            # Text search filter component
├── number_range_filter.rs    # Numeric range filter with slider
├── date_range_filter.rs      # Date range filter with calendar
├── faceted_filter.rs         # Multi-select checkbox filter
└── table_status_bar.rs       # Status display component
```

## Core Trait

### TableFilterComponent

All filter components implement this trait:

```rust
pub trait TableFilterComponent: Sized {
    type Value: FilterValue;
    const FILTER_TYPE: FilterType;
    
    fn new(cx: &mut WindowContext) -> Entity<Self>;
    fn value(&self) -> Self::Value;
}
```

This provides a uniform interface for:
- Creating filter entities in the GPUI context
- Retrieving current filter values
- Identifying filter types

## Filter Components

### TextFilter

**File:** `text_filter.rs`

A debounced text input for string filtering.

**Features:**
- 300ms debounce to avoid excessive updates
- Configurable validation (alphabetic, numeric, alphanumeric, custom)
- Clear button
- Placeholder text support

**Extension Trait:**
```rust
pub trait TextFilterExt {
    fn alphabetic_only(self) -> Self;
    fn numeric_only(self) -> Self;
    fn alphanumeric_only(self) -> Self;
    fn validate(self, validator: fn(&str) -> bool) -> Self;
}
```

**Value Type:** `String`

### NumberRangeFilter

**File:** `number_range_filter.rs`

A dual-input range filter with optional slider.

**Features:**
- Min/max input fields
- Synchronized slider control
- 300ms debounce
- Configurable range bounds and step

**Extension Trait:**
```rust
pub trait NumberRangeFilterExt {
    fn range(self, min: Decimal, max: Decimal) -> Self;
    fn step(self, step: Decimal) -> Self;
}
```

**Value Type:** `(Option<Decimal>, Option<Decimal>)`

### DateRangeFilter

**File:** `date_range_filter.rs`

A date range picker with calendar UI.

**Features:**
- Calendar-based date selection
- Start/end date inputs
- Visual date range highlighting

**Value Type:** `(Option<NaiveDate>, Option<NaiveDate>)`

### FacetedFilter

**File:** `faceted_filter.rs`

A multi-select filter for enum/boolean values.

**Features:**
- Checkbox list with labels
- Optional item counts
- Optional icons per option
- Optional search within options

**Extension Trait:**
```rust
pub trait FacetedFilterExt {
    fn searchable(self) -> Self;
}
```

**Value Type:** `HashSet<T>` where `T: Filterable`

## Component Lifecycle

```
1. Entity created via TableFilterComponent::new()
   ↓
2. Component renders UI in GPUI
   ↓
3. User interacts with filter controls
   ↓
4. Debounce timer fires (if applicable)
   ↓
5. on_change callback invoked
   ↓
6. Parent reads value via TableFilterComponent::value()
   ↓
7. Filter applied to table data
```

## TableStatusBar

**File:** `table_status_bar.rs`

A status display component (not a filter).

**Features:**
- Row count display
- Loading state indicator
- End-of-file status
- Customizable labels

**Builder Pattern:**
```rust
TableStatusBar::new()
    .row_count(100)
    .loading(false)
    .eof(true)
    .row_count_label("Rows:")
```

## Integration with Generated Code

The derive macro generates `FilterEntities` structs that hold `Entity<T>` for each filter:

```rust
// Generated by #[derive(GpuiTable)]
pub struct FooFilterEntities {
    pub name: Entity<TextFilter>,
    pub price: Entity<NumberRangeFilter>,
    pub category: Entity<FacetedFilter<Category>>,
}
```

The `FilterEntitiesExt::build()` method creates these entities:

```rust
impl FilterEntitiesExt<FooFilterValues> for FooFilterEntities {
    fn build(cx: &mut WindowContext) -> Self {
        Self {
            name: TextFilter::new(cx),
            price: NumberRangeFilter::new(cx).range(0.into(), 1000.into()),
            category: FacetedFilter::new(cx),
        }
    }
}
```

## Styling

Components use `gpui-component` primitives:
- `Input` for text fields
- `Slider` for range sliders
- `Checkbox` for faceted options
- `Calendar` for date selection

Styling follows the GPUI theming system.

## Dependencies

- `gpui` - Core UI framework

- `gpui-table-core` - Core traits and types
